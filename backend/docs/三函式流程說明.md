# 三函式流程說明：analyze_segment、infer_segment_yolo、get_yolo_model

本文檔詳細說明影片分析流程中三個核心函式的職責、流程與相依關係。

---

## 一、整體呼叫流程圖

```
run_full_pipeline(seg_files, ...)
    │
    └─► 對每個 segment_xxx.mp4 並行呼叫：
            │
            └─► analyze_segment(req)
                    │
                    ├─► [階段 1] VLM 分析（依 model_type）
                    │       ├─ qwen   → infer_segment_qwen()
                    │       ├─ gemini → infer_segment_gemini()
                    │       └─ moondream → infer_segment_moondream()
                    │
                    └─► [階段 2] YOLO + ReID（必執行）
                            │
                            └─► infer_segment_yolo(seg_path, labels, every_sec, score_thr)
                                    │
                                    ├─► get_yolo_model()      ← 取得 YOLO-World 模型
                                    ├─► get_reid_model()      ← 取得 ReID 模型
                                    ├─► cv2.VideoCapture 讀幀
                                    ├─► model.predict() 推論
                                    ├─► frame[y1:y2, x1:x2] 裁剪
                                    ├─► cv2.imwrite() 存檔
                                    └─► generate_reid_embeddings_batch() 特徵提取
```

---

## 二、get_yolo_model() 詳解

**檔案：** `src/core/model_loader.py`

**職責：** 以單例模式取得或初始化 YOLO-World 模型，並做健康檢查，避免使用殭屍模型。

### 流程概覽

1. **健康檢查**：若全域已有模型，檢查其是否仍有效  
   - 檢查 `.model` 是否存在且非 `None`  
   - 嘗試存取 `.model.names` 驗證可用性  
   - 若發現殭屍模型，則將全域變數設為 `None` 以觸發重載  

2. **模型載入**：僅在 `_yolo_world_model is None` 時執行  
   - 依優先順序決定權重路徑：  
     - 環境變數 `YOLO_WEIGHTS_PATH`  
     - `/app/models/yolov8s-world.pt`  
     - 當前目錄 `yolov8s-world.pt`（可能觸發下載）  
   - 使用 `YOLOWorld(model_path)` 建立模型  
   - 呼叫 `set_classes(["person"])` 觸發內部初始化  

3. **回傳**：回傳全域 YOLO 模型實例  

### 為什麼需要健康檢查？

- GPU 記憶體不足時，底層模型可能被釋放，但 Python 物件仍存在  
- 存取 `.model` 或 `.model.names` 會拋錯，稱為「殭屍模型」  
- 透過健康檢查及早重載，避免推論時崩潰

## 三、infer_segment_yolo() 詳解

**檔案：** `src/services/analysis_service.py`

**職責：** 對單一影片片段進行 YOLO 物件偵測、裁剪並存檔，同時產生 ReID 特徵。

### 流程概覽（6 階段）

| 階段 | 內容 |
|------|------|
| 1 | 取得 YOLO、ReID 模型 |
| 2 | 建立輸出目錄 `{seg_dir}/yolo_output/object_crops/` |
| 3 | 設定 YOLO 偵測類別與設備（含殭屍/設備錯誤處理） |
| 4 | 開啟影片，計算取樣間隔 `interval = max(1, fps * every_sec)` |
| 5 | 批次讀幀 → YOLO 推論 → 依 bbox 裁剪 → 存檔 → ReID 特徵 |
| 6 | 統計並回傳 `detections`、`crop_paths`、`object_count` |

### 關鍵參數

- **seg_path**：影片片段路徑（如 `segment_000.mp4`）  
- **labels**：逗號分隔的類別名稱（如 `"person,car,motorcycle"`）  
- **every_sec**：取樣頻率（秒），例如 2.0 表示每 2 秒取一幀  
- **score_thr**：信心門檻（0.0–1.0），低於此值的偵測會被過濾  

### 裁剪邏輯

```python
# 從 YOLO 結果取得邊界框 [x1, y1, x2, y2]
xyxy = [int(round(float(c))) for c in box.xyxy[0].tolist()]
x1, y1, x2, y2 = xyxy

# numpy 切片裁剪（OpenCV 影像為 HWC，故 [y1:y2, x1:x2]）
crop = frame[y1:y2, x1:x2]
cv2.imwrite(str(crop_path), crop)
```

### 輸出結構

```python
{
    "total_detections": int,       # 總偵測數
    "detections": [...],           # 每筆含 timestamp, label, score, box, frame, path, reid_embedding
    "crop_paths": [...],           # 同上，供 main.py 寫入 DB
    "object_count": {"person": 5, "car": 3, ...},
    "total_frames_processed": int  # 實際處理的幀數
}
```

## 四、analyze_segment() 詳解

**檔案：** `src/services/analysis_service.py`

**職責：** 對單一影片片段進行完整分析，結合 VLM（事件偵測 + 摘要）與 YOLO（物件偵測 + 裁剪 + ReID）。

### 流程概覽（2 階段）

| 階段 | 內容 |
|------|------|
| 1 | 依 `req.model_type` 選擇 VLM：`qwen` / `gemini` / `moondream` |
| 2 | 不論 VLM 類型，一律執行 YOLO + ReID，將結果放入 `raw_detection["yolo"]` |

### 輸入 req 關鍵欄位

- **segment_path**：片段路徑  
- **model_type**：`"qwen"` | `"gemini"` | `"moondream"` | `"yolo"`  
- **qwen_model**：實際模型名稱（如 `qwen3-vl:8b`）  
- **event_detection_prompt**：事件偵測 prompt  
- **summary_prompt**：摘要產生 prompt  
- **yolo_labels**：YOLO 偵測類別  
- **yolo_every_sec**：YOLO 取樣頻率  
- **yolo_score_thr**：YOLO 信心門檻  

### 輸出結構

```python
{
    "segment": "segment_000.mp4",
    "time_range": "00:00:00 - 00:00:10",
    "duration_sec": 10.0,
    "success": True/False,
    "time_sec": 12.5,
    "parsed": {
        "frame_analysis": {...},    # 標準化 VLM 事件分析
        "summary_independent": "..." # VLM 摘要文字
    },
    "raw_detection": {
        "yolo": {...}                # infer_segment_yolo 的回傳值
    },
    "error": None 或 "錯誤訊息"
}
```

### 重要設計

1. **YOLO 必定執行**：即便使用 Qwen/Gemini/Moondream，仍會跑 YOLO，以取得物件裁剪與 ReID，供後續以圖搜圖使用。  
2. **raw_detection["yolo"]**：`_save_results_to_postgres` 會依此 key 取得 YOLO 結果並寫入 `object_crops` 表。  
3. **錯誤合併**：若 VLM 與 YOLO 都有錯誤，會合併到 `result["error"]`。  

---

## 五、三函式相依關係

```
analyze_segment(req)
    │
    ├─► infer_segment_qwen / infer_segment_gemini / infer_segment_moondream  （VLM）
    │
    └─► infer_segment_yolo(seg_path, labels, every_sec, score_thr)
            │
            ├─► get_yolo_model()      ← model_loader
            ├─► get_reid_model()     ← model_loader
            └─► generate_reid_embeddings_batch()
```

- **get_yolo_model**：被 **infer_segment_yolo** 呼叫，負責 YOLO 模型的生命週期。  
- **infer_segment_yolo**：被 **analyze_segment** 呼叫，是單一片段物件偵測與裁剪的核心。  
- **analyze_segment**：被 **run_full_pipeline** 的 `process_one` 對每個片段呼叫，是單一片段分析的入口。  

---

## 六、資料流總覽

```
影片片段 (segment_000.mp4)
    │
    ▼
analyze_segment
    │
    ├─► VLM：取幀 → 送模型 → 得到事件分析 + 摘要
    │
    └─► infer_segment_yolo
            │
            ├─► get_yolo_model() 取得模型
            ├─► 每 every_sec 秒取一幀
            ├─► YOLO predict → 得到 boxes
            ├─► frame[y1:y2, x1:x2] 裁剪
            ├─► cv2.imwrite 存到 yolo_output/object_crops/
            └─► ReID 特徵 → 附加到每筆 detection
    │
    ▼
result（含 parsed + raw_detection["yolo"]）
    │
    ▼
_save_results_to_postgres → object_crops 表
```
